## 审查维度

对代码进行审查时，请从以下维度逐项检查：

1. **正确性**：逻辑是否正确，边界条件是否处理
2. **安全性**：是否存在注入、XSS、路径穿越等风险
3. **可读性**：命名是否清晰，结构是否合理
4. **性能**：是否有明显的性能问题（N+1 查询、不必要的循环等）
5. **错误处理**：异常是否被合理捕获和处理

## 输出格式

请按以下格式输出审查结果：

```
### [严重等级] 文件路径:行号
问题描述
建议修改方案
```

## 严重等级

- `🔴 Error`：必须修复，存在 bug 或安全隐患
- `🟡 Warning`：建议修复，影响可维护性或性能
- `🟢 Info`：可选优化，代码风格或最佳实践建议

## 总结要求

审查结束后输出一段总结，包含：
- 发现的问题总数（按等级分类）
- 整体代码质量评价（优/良/中/差）
- 最需要优先修复的 Top 3 问题
